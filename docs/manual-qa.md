# 手工验收清单

## 验收目标

确认桌面 Agent MVP 的端到端主链路可用：热键唤起、输入 prompt、进入 Agent Core、LLM 返回结果、气泡反馈显示。

## 验收前提

- 已完成依赖安装
- 已通过 `swift build`
- Web bundle 已生成

## 验收步骤

1. 启动桌面应用。
2. 确认主界面展示热键状态文案。
3. 按全局热键唤起输入框。
4. 输入一段用户主动发起的请求并提交。
5. 观察输入框关闭，气泡区出现用户消息和 Agent 回复。
6. 如当前环境已接入选区捕获，先选中文本再唤起，确认预填内容与选区一致。
7. 如 LLM 或工具返回错误，确认错误会以气泡形式展示，而不是静默失败。

## 通过标准

- 热键可唤起或关闭输入框
- 提交 prompt 后能进入一次完整运行
- 结果以气泡形式反馈到 UI
- 错误路径有可见反馈
