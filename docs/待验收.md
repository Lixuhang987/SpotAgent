- **验收结果**：第一版插件目录固定为 `~/.spotAgent/plugins/<plugin-id>/plugin.json`；manifest 声明 `id / name / version / enabled / tools[]`，每个 tool 声明 `name / description / inputSchema / command / timeoutMs / permissions`；plugin tool 不允许覆盖 builtin，重复 plugin tool 会禁用并记录原因；运行时复用 `tools.allowlist / tools.denylist` 做启停；插件作为本地受信任子进程通过 JSON stdin/stdout 执行，不提供 OS 级沙箱；command 真实路径必须留在插件目录内，stdout / stderr 各有 1 MiB 上限；非 0 exit、invalid JSON、timeout 与输出超限作为 tool error 返回，不拖垮 agent-server；workspace 权限声明复用 workspace 路径校验，插件只收到校验后的 workspaceRoot / absolutePath；权限记忆 hash 包含 toolName，`file.write` allow 不会静默放行 `plugin.*`。测试覆盖 manifest 解析、冲突、denylist、执行成功、崩溃 / 超时、输出超限、plugin command symlink 越界拦截、workspace 路径越界拦截与权限隔离。
